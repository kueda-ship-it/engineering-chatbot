# Engineering Chatbot 仕様書

**バージョン：** 1.0
**作成日：** 2026-04-02

---

## 1. システム概要

社内エンジニアおよび外部協力会社向けのAIチャットボット。
n8nワークフローエンジン経由でClaude API（またはOllama）に接続し、手順案内と社内システム（IRes/FC）のステータス取得を提供する。

---

## 2. 技術スタック

| 領域 | 技術 |
|------|------|
| フロントエンド | React 19 + TypeScript + Vite 8 |
| スタイリング | Tailwind CSS v4 |
| 状態管理 | TanStack Query v5 |
| DB | Supabase (PostgreSQL) |
| ワークフロー | n8n |
| AI | Claude API (Haiku) / Ollama (ハイブリッド) |
| アイコン | Lucide React |

## 4. DBスキーマ

### chat_sessions
| カラム | 型 | 説明 |
|--------|-----|------|
| id | uuid PK | セッションID |
| user_id | uuid FK | auth.users参照 |
| title | text | セッションタイトル |
| created_at | timestamptz | 作成日時 |
| updated_at | timestamptz | 更新日時（自動更新） |

### messages
| カラム | 型 | 説明 |
|--------|-----|------|
| id | uuid PK | メッセージID |
| session_id | uuid FK | chat_sessions参照 |
| role | text | 'user' or 'assistant' |
| content | text | メッセージ本文 |
| created_at | timestamptz | 作成日時 |

## 5. n8n Webhook I/F

### リクエスト (POST)
```json
{
  "message": "M-001のステータスを確認して",
  "session_id": "uuid",
  "user_role": "internal",
  "history": [
    { "role": "user", "content": "..." },
    { "role": "assistant", "content": "..." }
  ]
}
```

### レスポンス
```json
{
  "reply": "機番 M-001 のステータス: 対応中",
  "intent": "status"
}
```

### intent 種別
| 値 | 意味 |
|----|------|
| faq | 手順案内・FAQ応答 |
| status | FCステータス取得 |
| log | ログ取得（Phase 2） |
| escalation | 担当者エスカレーション |
| unknown | 不明（デフォルト応答） |

---

## 6. ユーザー権限

| role | 機能 |
|------|------|
| internal | 全機能（手順案内 + ステータス取得 + ログ参照） |
| partner | 手順案内のみ |
